# ThereminQ Holoqubed Concept

This directory contains conceptual prototypes for ThereminQ Holoqubed.

## Files

*   `gguf_vs_holo_divergences.py`: Accuracy Harness. Runs a dense GGUF reference model side-by-side with the sparse `.holo` engine to measure mathematical divergence caused by the SiLU sparsity threshold. It leverages the `llama.cpp` python bindings to fetch dense logits, executes a mocked O(log N) sparse spatial lookup via the `HoloQueryPlanner`, and computes the Top-1 Accuracy Match and Mean Squared Error (MSE) to tune the pruning threshold during the offline forge phase.
*   `holo_generate_hf.py`: Text Generation Engine. It monkey-patches a Hugging Face `PreTrainedModel` by traversing its architecture and replacing dense `nn.Linear` layers with custom `HoloLinear` PyTorch modules. These modules are backed by an OpenCL SpMV engine, allocating static Zero-Copy buffers per layer, transferring PyTorch tensors to OpenCL space, and evaluating them using the `spmv_holo_weights` OpenCL kernel. It intercepts the forward pass dynamically, making it compatible with `AutoModelForCausalLM` and `AutoTokenizer` execution.
*   `holoqubed_prototype.py`: PyOpenCL Prototype. A sparse, holographic AI inference engine bypassing dense matrix math via O(1) spatial queries and FP16 Rapid Packed Math. Targeting AMD Radeon Pro V340 (Vega 10) hardware via Mesa Rusticl, it encodes floating-point activation thresholds into 1D spatial coordinates using a custom bit-interleaving scheme (bitwise XOR) forming a Z-order (Morton) curve, avoiding traditional Hilbert curve algorithms. It simulates a 320GB System RAM dictionary using Zero-Copy bindings and executes a custom Rapid Packed Math (`half2`) OpenCL kernel to evaluate the Tesseract KV Cache via a Scatter-Gather pattern and hardware SiLU activation.
